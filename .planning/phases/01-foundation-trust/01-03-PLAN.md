## Objective

Implement protected route infrastructure, session management, logout functionality, and user profile access. Users stay logged in across sessions and can log out from any page. The protected layout provides navigation and session context.

## Verification Criteria

1. **Protected Routes**: Unauthenticated users cannot access protected pages
2. **Session Persistence**: User stays logged in after browser refresh
3. **Logout**: User can log out from header dropdown on any page
4. **Navigation**: Header and nav components render correctly
5. **User Context**: Client components can access user state via hooks
