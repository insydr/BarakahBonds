## Verification Criteria

1. **Login Flow**: User can enter credentials and receive appropriate feedback
2. **Registration Flow**: User can create account with validation
3. **Email Verification**: Callback route properly exchanges tokens
4. **Error Handling**: All errors use generic, non-enumerating messages
5. **Discrete Branding**: No relationship-related terminology visible
6. **Session Persistence**: Redirects work correctly for authenticated/unauthenticated users

## Must Haves (Goal-Backward Verification)

- [ ] Login page with email/password form (AUTH-01)
- [ ] Registration page with 12+ char password requirement (AUTH-01)
- [ ] Email verification flow with callback handling (AUTH-01)
- [ ] Generic error messages prevent account enumeration (AUTH-01)
- [ ] Session persistence via Supabase SSR (AUTH-02)
- [ ] Discrete branding throughout all auth pages (AUTH-07)
