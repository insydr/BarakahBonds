## Objective

Implement the complete couple linking flow with invitation code generation, acceptance, and approval. Add privacy settings including "burn after reading" toggle. Ensure all UI maintains discrete branding with generic notification text.

### Task 2: Create Couple Server Actions

<read_first>
- .planning/phases/01-foundation-trust/01-RESEARCH.md (Section 4.3)
- lib/supabase/server.ts
- utils/invitation-code.ts
</read_first>

<action>
Create `actions/couple.ts` with `'use server'` directive. Export five async functions:
1. `generateInvite()` - Creates couples record with invitation_code, sets status to 'pending', expires in 7 days
2. `acceptInvite(code: string)` - Validates code, sets partner_2_id, returns couple ID
3. `approveLink(coupleId: string)` - Only partner_1_id can call, sets status to 'active'
4. `rejectLink(coupleId: string)` - Deletes pending couple record
5. `disconnectPartner()` - Sets status to 'inactive'

All functions check user authentication and return typed results with success/error.
</action>

<acceptance_criteria>
- `actions/couple.ts` file exists

- File exports `generateInvite` async function
- File exports `acceptInvite` async function
- File exports `approveLink` async function
- File exports `rejectLink` async function
- File exports `disconnectPartner` async function
- `generateInvite` creates record in `couples` table with status 'pending'
- `acceptInvite` validates invitation code and expiry
- `approveLink` updates status to 'active'
- `disconnectPartner` updates status to 'inactive'
</acceptance_criteria>

## Verification Criteria

1. **Invitation Generation**: User can generate and copy 8-character code
2. **Invitation Acceptance**: Partner can enter code and request link
3. **Approval Flow**: Initiator can approve/reject link requests
4. **Disconnection**: Either partner can disconnect with confirmation
5. **Privacy Settings**: Burn after reading toggle works and saves
6. **Discrete Branding**: All UI uses generic terminology
