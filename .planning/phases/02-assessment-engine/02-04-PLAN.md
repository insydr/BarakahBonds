## Objective

Implement PDF report generation for the Couple's Report. This professional document is designed for Imam/mentor review and includes category scores, red flag summaries, and discussion prompts with Islamic context. Reports are generated on-demand and stored temporarily.

Install `@react-pdf/renderer` for server-side PDF generation:

```bash
npm install @react-pdf/renderer
```

This library allows React components to render as PDF documents, providing:
- Component-based PDF templates
- Server-side rendering
- Custom fonts and styles
- Image support

### Task 8: Create PDF API Route

<read_first>
- app/api/ directory structure
- .planning/phases/02-assessment-engine/02-CONTEXT.md (D-14)
</read_first>

<action>
Create `app/api/reports/[id]/route.ts`:

1. GET endpoint:
   - Authenticate user
   - Verify user owns the assessment (or is partner)
   - Check both partners have completed
   - Generate PDF on-demand
   - Return PDF with appropriate headers

2. Response headers:
   - `Content-Type: application/pdf`
   - `Content-Disposition: attachment; filename="couple-report.pdf"`

3. Error handling:
   - 401 if not authenticated
   - 403 if not authorized (not your assessment)
   - 404 if assessment not found
   - 400 if assessment incomplete
</action>

</read_first>

<action>
Create `actions/report.ts` with:

1. `generateReportAction(assessmentId: string)`:

   - Generate PDF using `generateCoupleReport`
   - Store temporarily (buffer in memory, optional: S3/Supabase storage)
   - Return download URL or blob

2. `getReportStatusAction()`:
   - Check if couple can generate report

3. Integration with assessment completion:
   - After user completes, check if partner also complete
   - Show "Generate Report" button if ready
</action>
