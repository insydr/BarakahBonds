## Objective

Create the individual results view that works in solo mode (primary experience) and enhances with partner comparison when both complete. Users see their scores immediately upon completion, with actionable insights and optional partner invitation.

### Task 7: Create Results Server Actions

<read_first>
- actions/assessment.ts
- lib/database/assessments.ts
</read_first>

<action>
Create `actions/results.ts` with:

1. `getResultsAction()`:
   - Fetch current user's assessment with scores
   - Return scores, flags, completion status
   - Return null if not completed

2. `getPartnerResultsAction()`:
   - Check if user has linked partner
   - Check if partner has completed assessment
   - Return partner's scores (not individual responses)
   - Return compatibility score

3. `sendPartnerReminderAction()`:
   - Send notification to partner
   - Rate-limited (max 1 per 24 hours)
   - Use discrete notification text

4. Integration:
   - Call scoring functions from lib/assessment/scoring.ts
   - Use couple functions from Phase 1
</action>
