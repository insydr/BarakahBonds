## Objective

Implement the scoring algorithm for the Assessment Engine. This includes category-based scoring (Deen, Dunya, Aila, Nafs), red flag detection with two severity levels, and couple compatibility scoring. The algorithm is implemented as pure functions for testability and reproducibility.

<action>
Create `lib/assessment/scoring.ts` with:

1. `calculateSectionScore(responses: Response[], section: string): SectionScore`:
   - Filter responses for the section
   - Convert Likert values to 0-100 scale (1→0, 5→100)
   - Calculate average for answered questions
   - Track skipped count
   - Return SectionScore object

2. `calculateOverallScore(sectionScores: SectionScore[]): number`:
   - Apply weights from SECTION_WEIGHTS constant

   - Return weighted average (0-100)

3. `calculateAssessmentScore(allResponses: Response[]): AssessmentScore`:
   - Calculate score for each section
   - Calculate overall weighted score
   - Return complete AssessmentScore object

All functions should be pure (no side effects) and exportable.
</action>

</read_first>

<action>
Create `lib/assessment/compatibility.ts` with:

1. `calculateSectionAlignment(responses1: Response[], responses2: Response[], section: string): SectionAlignment`:
   - Compare responses for matching questions
   - Calculate alignment percentage (how similar)
   - Identify divergent questions (>2 point difference on Likert)

3. `generateDiscussionAreas(divergentResponses: DivergentResponse[]): DiscussionArea[]`:
   - Transform divergent responses into discussion prompts
   - Include relevant Islamic guidance for each area
   - Add suggested conversation starters

4. Export types used: `SectionAlignment`, `DivergentResponse`
</action>
